这里必须要先明白暂存区的概念，一般项目开发，代码不可能刚好一次提交完毕，需要将代码暂存起来，利用暂存区去提交代码（stash）

1.暂存代码防止干扰：（本地修改提交合并时，切换分支常用到）
1）git add . ，代码更改添加到暂存
2）git stash，暂存所有修改，注意不add则不生效
3）git pull，拉取最新代码
4）git stash apply，应用暂存区的代码
此时若有冲突，解决冲突即可

2.部分提交代码：
1）git add file1... ，多次添加需要提交的代码到暂存区
2）git stash -k -u，隐藏其他修改，此时status只能看到添加的暂存代码
3）git commit -m ''，提交代码到本地仓库
4）git stash pop，恢复之前隐藏的代码修改

实际在开发中，其实很推荐图形界面的git管理工具，比如sourcetree，甚至是git自带的图形界面，对这种复杂操作来说更加友好，也很方便

理解上静态属性可以参考静态方法：
定义在类上的属性，属于类自身，不属于类的实例化对象，且只能由类本身调用
（注意和实例属性区分（实例属性定义在this上，属于类的实例化对象））

不过ES6规定，class内部只有静态方法，没有静态属性，所以目前只能这么写：
class Foo { }
Foo.prop = 1;
Foo.prop    // 1
这么写不是很优雅，所以后面有提案，类似静态方法的写法：
将静态属性定义在实例属性之前，并用static修饰
class Foo {
    static prop = 1;
    constructor() {
        this.name = 'tom';
    }
}
注意只是提案，还没有支持。

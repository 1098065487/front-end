一个最明显的，可以让页面动画在移动设备上运行的更快一些

导致js效率低的两大原因：操作Dom，使用页面动画
会导致浏览器不断的执行重排和重绘，对于pc端浏览器，浏览器可用内存比较大，还可以接受，但是移动端浏览器内存普遍不够大，js动画正式通过操作dom的css实现动画的，在移动端很影响页面运行

css动画对比js动画优点：
1）不占用js主线程
2）可以利用硬件加速
3）浏览器可对动画做优化（元素不可见时不动画，减少对FPS的影响）
缺点：浏览器对渲染的批量异步化处理让动画难以控制（不太能理解）

css3动画和js模拟动画对比：
1）css 3D动画无法在js中实现
2）css 2D矩阵动画效率高于js利用margin,left,top等模拟的矩阵动画
3）css3其他常规动画属性效率均低于js模拟动画（height, width, opacity, border-width, color..）
